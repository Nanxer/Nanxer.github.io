Docker搭建GZCTF平台

约 900 字大约 3 分钟

GZCTFDocker

2025-02-24

tips：需要wsl环境（不用安装linux发行版，docker-desktop自带有，但是wsl环境要能用）

1.下载docker-desktop

在官网docker根据自己win的cpu种类选择amd64或者arm64，然后安装

2.docker换源

由于“墙”的缘故，大部分docker hub上的资源在国内直接访问不到/很慢，所以一般都会换成国内镜像，可以使用阿里云提供的服务：阿里云登录 - 欢迎登录阿里云，安全稳定的云计算服务平台

同时建议收藏此网址，可以使用“制品中心”的国内镜像来替换dockerfile中同名国外镜像来加速下载

在docker的“设置”->“Docker Engine”->添加registry-mirrors（在上方的镜像加速器里的Windows选项下有详细过程）

3.配置GZCTF

可以研究一下官方文档GZ:CTF，内有详细安装文档及配置过程

新建文件：appsettings.json(可以仅修改##1##处，并记住，要有一定复杂性，下一个文件要用到)

{
  "AllowedHosts": "*",
  "ConnectionStrings": {
    "Database": "Host=db:5432;Database=gzctf;Username=postgres;Password=##1##"
  },
  "EmailConfig": {
    "SenderAddress": "",
    "SenderName": "",
    "UserName": "",
    "Password": "",
    "Smtp": {
      "Host": "localhost",
      "Port": 587
    }
  },
  "XorKey": "<Your XOR_KEY>",
  "ContainerProvider": {
    "Type": "Docker", // or "Kubernetes"
    "PortMappingType": "Default", // or "PlatformProxy"
    "EnableTrafficCapture": false,
    "PublicEntry": "xxx.xxx.xxx.xxx", // or "xxx.xxx.xxx.xxx"
    // optional
    "DockerConfig": {
      "SwarmMode": false,
      "Uri": "unix:///var/run/docker.sock"
    }
  },
  "RegistryConfig": {
    "UserName": "",
    "Password": "",
    "ServerAddress": ""
  },
  "CaptchaConfig": {
    "Provider": "None", // or "CloudflareTurnstile" or "HashPow"
    "SiteKey": "<Your SITE_KEY>",
    "SecretKey": "<Your SECRET_KEY>"
  },
  "ForwardedOptions": {
    "ForwardedHeaders": 7,
    "ForwardLimit": 1,
    "TrustedNetworks": ["192.168.12.0/8"]
  }
}

compose.yml(##2##处的密码要包含大小写和数字/比如Admin123，否则默认管理员admin不能直接加入数据库，需要另外创建/较为麻烦；##1##与上一文件相同)

services:
  gzctf:
    image: registry.cn-shanghai.aliyuncs.com/gztime/gzctf:develop
    restart: always
    environment:
      - "GZCTF_ADMIN_PASSWORD=<##2##>"
      # choose your backend language `en_US` / `zh_CN` / `ja_JP` ...
      - "LC_ALL=zh_CN.UTF-8"
    ports:
      - "80:8080"
    volumes:
      - "./data/files:/app/files"
      - "./appsettings.json:/app/appsettings.json:ro"
      # - "./kube-config.yaml:/app/kube-config.yaml:ro" # this is required for k8s deployment
      - "/var/run/docker.sock:/var/run/docker.sock" # this is required for docker deployment
    depends_on:
      - db

  db:
    #image: postgres:alpine
    image: anolis-registry.cn-zhangjiakou.cr.aliyuncs.com/openanolis/postgres:10.21-8.6
    restart: always
    environment:
      - "POSTGRES_PASSWORD=<##1##>"
    volumes:
      - "./data/db:/var/lib/postgresql/data"

通过制品中心换一个postgres的国内镜像

4.安装

在appsettings.json和compose.yml所在目录下，打开终端，使用命令

docker compose up -d

进行启动

现在打开docker，看到有端口/网站可以访问了

登陆默认账号admin，密码为自己设置的含大小写字母和数字，比如Admin123

创建新的比赛部署题目

创建测试容器，并一定一定“保存更改”(容器镜像根据自己拉取的使用)

注意，这里的public_entry要在appsettings.json中设置，不设置就手动输入端口

(题目来源：GitHub - XDSEC/MoeCTF_2024Challenges/Web/ez_http，并将init.sh的FLAG修改为GZCTF_FLAG，从而实现动态flag/参考GZ:CTF/动态flag，在目录下使用docker build -t ez_http .制作名为ez_http的镜像来使用)

启用题目，就可以使用

其他题目部署类似/参考官方文档
